# RunPod

Alternatives Names: RunPod Cloud; RunPod Serverless
Resource Link: https://docs.runpod.io
Category: GPU Compute / Cloud
Programming Language: Python / REST API
Description: GPU cloud marketplace offering on-demand and spot GPU instances. Cheaper than AWS for raw GPU hours, with a GraphQL API for spinning up pods. Good for longer-running training jobs where Modal's per-second billing becomes costly.
Role in VC Assistant Infrastructure: Alternative or complement to Modal for long fine-tuning runs (multi-hour) or persistent vLLM inference endpoints for tenants with high and predictable inference volume.
Potential Alternatives (with main comparison): Modal: better developer experience and Python-native API but more expensive for long-running jobs. AWS SageMaker: enterprise support and compliance but 3-5x the cost. Lambda Labs: similar pricing but smaller GPU selection. CoreWeave: better for very large scale but higher minimum commitment.
Deployment Notes & File Structure: RunPod jobs are pod-based. Recommended structure:
infra/
  runpod/
    Dockerfile            # GPU training image
    train.py              # entry point script
    runpod_dispatch.py    # API client to launch pods
    pod_monitor.py        # poll status + collect results
    requirements.txt
Short File/Config Template: # runpod_dispatch.py
import runpod
runpod.api_key = "your-api-key"

pod = runpod.create_pod(
    name=f"finetune-{tenant_id}",
    image_name="your-ecr/vc-trainer:latest",
    gpu_type_id="NVIDIA A100 80GB PCIe",
    env={"TENANT_ID": tenant_id, "S3_BUCKET": bucket}
)
Stack & Framework Integration: Controlled via GraphQL API or Python SDK (runpod-python). Integrates with S3 for artifact storage. Can run any Docker image — use the same image as Modal jobs for consistency. Pair with a job queue (Celery/Redis) in FastAPI to dispatch training jobs.

# Stack & Framework Integration

Controlled via GraphQL API or Python SDK (runpod-python). Integrates with S3 for artifact storage. Can run any Docker image — use the same image as Modal jobs for consistency. Pair with a job queue (Celery/Redis) in FastAPI to dispatch training jobs.
