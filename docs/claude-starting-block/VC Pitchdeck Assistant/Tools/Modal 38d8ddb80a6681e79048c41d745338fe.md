# Modal

Alternatives Names: Modal Labs; Modal Cloud
Resource Link: https://modal.com/docs
Category: GPU Compute / Serverless
Programming Language: Python
Description: Serverless cloud platform for running GPU workloads on-demand. Spins up containers with GPU access in seconds and charges only for actual compute time used. No reserved instances.
Role in VC Assistant Infrastructure: Runs fine-tuning jobs (LoRA/QLoRA) triggered when a tenant accumulates enough labeled data, and serves vLLM inference endpoints for custom fine-tuned models per tenant.
Potential Alternatives (with main comparison): RunPod: cheaper raw GPU hours but no Python-native decorator API — more manual pod management. Replicate: simpler model serving but less control over training pipelines. AWS SageMaker: more mature but significantly higher ops overhead and cost. Google Vertex AI: better GCP integration but heavier setup.
Deployment Notes & File Structure: Single file per job type. Recommended structure:
infra/
  modal/
    finetune_job.py       # LoRA training function
    inference_server.py   # vLLM serving endpoint
    image_builder.py      # shared GPU Docker image def
    utils.py              # S3 upload helpers
Short File/Config Template: # finetune_job.py
import modal
app = modal.App("vc-finetune")
gpu_image = modal.Image.debian_slim().pip_install("transformers","peft","trl")

@app.function(gpu="A100", image=gpu_image, timeout=3600)
def finetune(tenant_id: str):
    # load base model, run LoRA fine-tune, push artifact to S3

Stack & Framework Integration: Integrates natively with Python via @app.function decorators. Works directly with HuggingFace Transformers, PEFT, TRL, and vLLM. Triggered from FastAPI background tasks (via Celery or direct async call). Artifacts pushed to S3 post-training.

# Stack & Framework Integration

Integrates natively with Python via @app.function decorators. Works directly with HuggingFace Transformers, PEFT, TRL, and vLLM. Triggered from FastAPI background tasks (via Celery or direct async call). Artifacts pushed to S3 post-training.

# Alternative Tools

RunPod: cheaper raw GPU hours but no Python-native decorator API — more manual pod management. Replicate: simpler model serving but less control over training pipelines. AWS SageMaker: more mature but significantly higher ops overhead and cost. Google Vertex AI: better GCP integration but heavier setup.
