# ECS

Alternatives Names: Elastic Container Service; AWS ECS; ECS Fargate
Resource Link: https://docs.aws.amazon.com/ecs
Category: Container Orchestration
Programming Language: JSON / YAML (task defs) / Python (boto3)
Description: AWS-native container orchestration without Kubernetes. Runs Docker containers via EC2 or Fargate (serverless). Tighter AWS integration than EKS with significantly lower operational overhead. No cluster nodes to manage with Fargate.
Role in VC Assistant Infrastructure: Recommended starting container platform for early-stage team. Runs FastAPI API service, Celery workers for async document processing, and scheduled fine-tuning trigger tasks. Migrate to EKS when per-tenant namespace isolation or custom scheduling is needed.
Potential Alternatives (with main comparison): EKS: more powerful and portable but higher ops burden — migrate to EKS at scale. Google Cloud Run: similar serverless container model but GCP-only. Railway / Render: simpler PaaS options for very early stage but less control and compliance. Fly.io: good developer experience but not enterprise-ready. Nomad (HashiCorp): lighter orchestrator, less ecosystem support than Kubernetes.
Deployment Notes & File Structure: Recommended structure:
infra/
  ecs/
    task-definitions/
      vc-api.json
      celery-worker.json
      doc-pipeline.json
    services/
      vc-api-service.tf   # Terraform ECS service resource
    ecr/
      build-push.sh       # Docker build + ECR push script
    appspec.yml           # CodeDeploy blue/green config
Short File/Config Template: # ecs-task-definition.json (excerpt)
{
  "family": "vc-api-task",
  "requiresCompatibilities": ["FARGATE"],
  "cpu": "1024", "memory": "2048",
  "containerDefinitions": [{
    "name": "vc-api",
    "image": "your-ecr/vc-api:latest",
    "portMappings": [{"containerPort": 8000}]
  }]
}
Stack & Framework Integration: Provisioned by Terraform (ecs_service module). Task definitions stored as JSON in the repo and registered via CLI or Terraform. Service auto-scaling via Application Auto Scaling. Logs shipped to CloudWatch Logs. Secrets injected from AWS Secrets Manager into task environment at runtime.

# Stack & Framework Integration

Provisioned by Terraform (ecs_service module). Task definitions stored as JSON in the repo and registered via CLI or Terraform. Service auto-scaling via Application Auto Scaling. Logs shipped to CloudWatch Logs. Secrets injected from AWS Secrets Manager into task environment at runtime.

# Alternative Tools

EKS: more powerful and portable but higher ops burden — migrate to EKS at scale. Google Cloud Run: similar serverless container model but GCP-only. Railway / Render: simpler PaaS options for very early stage but less control and compliance. [Fly.io](http://fly.io/): good developer experience but not enterprise-ready. Nomad (HashiCorp): lighter orchestrator, less ecosystem support than Kubernetes.
