# PEFT

Alternatives Names: Parameter-Efficient Fine-Tuning; HF PEFT
Resource Link: https://huggingface.co/docs/peft
Category: Fine-Tuning Library
Programming Language: Python
Description: Hugging Face library implementing LoRA, QLoRA, Prefix Tuning, IA3, and other efficient adaptation methods. Provides the get_peft_model() wrapper and adapter save/load utilities. The standard Python interface for all LoRA-based fine-tuning workflows.
Role in VC Assistant Infrastructure: Applied in Modal/RunPod training jobs to attach LoRA adapters to base models, run training, and save adapter weights to S3. Also used at inference time to load the correct tenant-specific adapter on top of the shared base model in vLLM.
Potential Alternatives (with main comparison): Unsloth: faster LoRA training (2x speed, 60% less VRAM) but less flexible and fewer supported models — good for speed-critical pipelines. LlamaFactory: GUI-driven fine-tuning framework built on PEFT, easier for non-engineers. Axolotl: YAML-configured fine-tuning wrapper around PEFT, good for standardized training configs. TRL alone: focused on RLHF/reward modeling, not a PEFT replacement.
Deployment Notes & File Structure: PEFT is a library, not a service — no infra to deploy. Recommended usage structure:
training/
  peft_train.py           # adapter training entrypoint
  peft_inference.py       # load adapter for serving
  adapter_registry.py     # maps tenant_id to adapter S3 path
  eval/
    run_eval.py           # evaluate adapter quality post-training
Short File/Config Template: # peft_train.py
from peft import get_peft_model, LoraConfig
from transformers import AutoModelForCausalLM
base_model = AutoModelForCausalLM.from_pretrained("mistralai/Mistral-7B-v0.1")
peft_config = LoraConfig(r=16, lora_alpha=32, task_type="CAUSAL_LM")
model = get_peft_model(base_model, peft_config)
model.print_trainable_parameters()
Stack & Framework Integration: Direct dependency of LoRA and QLoRA workflows. Works with Transformers, TRL's SFTTrainer, and bitsandbytes. Adapters saved as safetensors files, versioned per tenant in S3 or HF Hub. At inference, model.load_adapter(tenant_path) hot-swaps the adapter without reloading base model weights.

# Stack & Framework Integration

Direct dependency of LoRA and QLoRA workflows. Works with Transformers, TRL's SFTTrainer, and bitsandbytes. Adapters saved as safetensors files, versioned per tenant in S3 or HF Hub. At inference, model.load_adapter(tenant_path) hot-swaps the adapter without reloading base model weights.
