# Langfuse

Alternatives Names: Langfuse OSS; LF Tracing; Open-source LangSmith
Resource Link: https://langfuse.com/docs
Category: LLM Observability
Programming Language: Python / TypeScript
Description: Open-source, self-hostable LLM observability platform. Provides traces, prompt versioning, evals, and cost tracking. Can be deployed inside your own AWS VPC — critical when clients prohibit their query data from leaving your controlled infrastructure.
Role in VC Assistant Infrastructure: Preferred over LangSmith for clients with strict data sovereignty requirements. Deployed inside AWS VPC. Tracks LLM traces per tenant with tenant-scoped dashboards for cost chargeback. Also manages prompt versions across dev/staging/prod environments.
Potential Alternatives (with main comparison): LangSmith: better UX and deeper LangChain integration but SaaS-only — eliminates it for data-sensitive clients. Helicone: proxy-based, simpler but less feature-rich and also SaaS. Arize Phoenix: stronger eval framework, self-hostable, good alternative. W&B Weave: self-hostable LLM tracing from Weights & Biases. OpenLLMetry: OpenTelemetry-based LLM instrumentation, outputs to any OTel backend.
Deployment Notes & File Structure: Self-hosted deployment. Recommended structure:
infra/
  langfuse/
    helm-values.yaml      # Langfuse Helm chart config
    postgres.tf           # RDS instance for Langfuse
    secrets.yaml          # K8s secret for DB credentials
app/
  llm/
    langfuse_client.py    # singleton Langfuse init
    decorators.py         # @observe() wrappers per function
    prompt_manager.py     # pull versioned prompts from Langfuse
Short File/Config Template: # langfuse_client.py
from langfuse import Langfuse
from langfuse.decorators import observe
langfuse = Langfuse(public_key="pk", secret_key="sk",
                    host="https://langfuse.internal.yourdomain.com")
@observe()
def screen_deck(tenant_id: str, deck_text: str):
    return llm.invoke(deck_text)
Stack & Framework Integration: Python SDK with @observe() decorator for non-LangChain code. LangChain integration via CallbackHandler. Self-hosted on EKS/ECS using official Docker image and PostgreSQL backend. Deployed via Helm chart on EKS. Tenant ID injected as trace metadata for filtering and cost allocation per client.

# Stack & Framework Integration

Python SDK with @observe() decorator for non-LangChain code. LangChain integration via CallbackHandler. Self-hosted on EKS/ECS using official Docker image and PostgreSQL backend. Deployed via Helm chart on EKS. Tenant ID injected as trace metadata for filtering and cost allocation per client.

# Alternative Tools

LangSmith: better UX and deeper LangChain integration but SaaS-only — eliminates it for data-sensitive clients. Helicone: proxy-based, simpler but less feature-rich and also SaaS. Arize Phoenix: stronger eval framework, self-hostable, good alternative. W&B Weave: self-hostable LLM tracing from Weights & Biases. OpenLLMetry: OpenTelemetry-based LLM instrumentation, outputs to any OTel backend.
